# Webアプリを作るための準備

## 今日何をしたのか

今日はいきなりアプリを作ったわけではない。

アプリを作れる環境を自分のMacに作った。

例えるなら、

料理を始める前に

* キッチンを用意する
* コンロを置く
* 包丁を買う

みたいな準備をした。

---

# Webアプリって何？

Webアプリとは、ブラウザで使うサービスのこと。

例えば

* ChatGPT
* YouTube
* Instagram
* X

など。

今回作ろうとしている

「Couple Quest」

もWebアプリ。

---

# 今日インストールしたもの

## ① Node.js

### 何者？

パソコンがプログラムを動かすためのエンジン。

車でいうエンジンみたいなもの。

### なぜ必要？

これがないと、

パソコンはWebアプリのプログラムを動かせない。

### 確認方法

```bash
node -v
```

実行すると

```text
v24.16.0
```

のような数字が表示される。

これは

「Node.jsが入っています」

という意味。

---

## ② npm

### 何者？

開発者向けのApp Store。

### なぜ必要？

便利な機能をダウンロードするため。

iPhoneで

App Storeからアプリを入れるのと同じ。

npmを使うと

* Next.js
* React
* Stripe

などをインストールできる。

### 確認方法

```bash
npm -v
```

---

## ③ Next.js

### 何者？

Webアプリを作るための土台。

### なぜ必要？

家を建てるとき、

毎回ゼロから設計図を書かない。

テンプレートを使う。

Next.jsも同じ。

Webアプリ開発用のテンプレート。

### 今日やったこと

```bash
npx create-next-app@latest couple-game
```

を実行した。

結果として

```text
couple-game
```

というフォルダが作られた。

このフォルダの中に

アプリの材料が入っている。

---

# localhostとは

今日開いたURL

```text
http://localhost:3000
```

### localhostとは？

自分のパソコンだけが見られるサイト。

まだ世界には公開されていない。

例

```text
Instagram
```

↓

世界中が見られる

---

```text
localhost
```

↓

自分だけが見られる

---

つまり今日は

世界公開前の練習用サイトを立ち上げた。

---

# Gitとは

### 何者？

セーブ機能。

ゲームのセーブデータと同じ。

例えば

今日の状態を保存

↓

明日壊した

↓

昨日の状態へ戻す

ことができる。

---

## Gitで使うコマンド

変更を管理対象にする

```bash
git add .
```

保存する

```bash
git commit -m "変更内容"
```

---

# GitHubとは

### 何者？

オンライン上のセーブデータ置き場。

### なぜ必要？

Gitだけだと

Macが壊れたら終わり。

GitHubに保存しておけば、

別のパソコンでも復元できる。

---

## GitHubへ保存する

```bash
git push
```

---

# 今日やったことまとめ

今日はアプリを作ったのではなく、

アプリを作るための工房を作った。

流れはこう。

```text
Mac

↓

Node.js
（プログラムを動かす）

↓

npm
（便利な機能を入れる）

↓

Next.js
（アプリの土台を作る）

↓

localhost
（自分だけが見られる状態で確認）

↓

Git
（履歴を保存）

↓

GitHub
（クラウドに保存）
```

ここまでで、

Webアプリ開発を始める準備が完了した。
