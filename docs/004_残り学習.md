# ターミナルでよく使ったコマンド

## このドキュメントの目的

今日の学習で実際に使ったコマンドをまとめる。

覚えるためというより、

「次に何か作る時に見返す用」

として残しておく。

---

# ターミナルとは

Macを文字で操作するためのアプリ。

普段はFinderで

* フォルダを開く
* ファイルを作る
* フォルダを移動する

などを行う。

ターミナルではそれを文字で操作できる。

---

# 今いる場所を確認する

```bash
pwd
```

例

```text
/Users/shogo/web-dev-notes
```

意味

今いるフォルダを表示する。

迷ったらまずこれを実行する。

---

# フォルダを移動する

```bash
cd フォルダ名
```

例

```bash
cd ~/web-dev-notes
```

意味

web-dev-notesフォルダへ移動する。

---

# ファイル一覧を確認する

```bash
ls
```

例

```text
README.md
docs
```

意味

現在のフォルダに何が入っているか確認する。

---

# フォルダを作成する

```bash
mkdir フォルダ名
```

例

```bash
mkdir docs
```

意味

docsフォルダを作成する。

---

# ファイルを作成する

```bash
touch ファイル名
```

例

```bash
touch docs/003_Webアプリを作るための準備.md
```

意味

空のファイルを作成する。

---

# Gitの状態を確認する

```bash
git status
```

意味

Gitが現在どのような状態か確認する。

例

```text
Untracked files:
```

新しいファイルを発見した状態。

---

# Gitに変更を登録する

```bash
git add .
```

意味

変更したファイルを管理対象に追加する。

例えるなら

「セーブする候補を選ぶ」

作業。

---

# 変更履歴を保存する

```bash
git commit -m "変更内容"
```

例

```bash
git commit -m "ターミナル基礎"
```

意味

現在の状態を履歴として保存する。

例えるなら

ゲームのセーブ。

---

# GitHubへ保存する

```bash
git push
```

意味

ローカルPCの履歴をGitHubへ送る。

例えるなら

オンラインにバックアップを保存する。

---

# 今日一番使った流れ

新しいドキュメント作成

↓

内容を書く

↓

```bash
git status
```

↓

```bash
git add .
```

↓

```bash
git commit -m "変更内容"
```

↓

```bash
git push
```

↓

GitHubに保存完了

---

```bash
pwd

git commit -m "..."
git push
```

まずはこの8個だけ覚えれば十分。
